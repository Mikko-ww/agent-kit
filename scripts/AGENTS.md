# Scripts 目录说明

本目录继承根目录 [../AGENTS.md](../AGENTS.md) 的全局规则，本文只补充脚本目录的组织约束。

## 1. 目录职责

- `scripts/dev/`：本地开发、调试、工作区辅助脚本。
- `scripts/release/`：版本发布、registry 同步、tag 编排等发布脚本。

## 2. 组织规则

- `scripts/` 根目录尽量只保留子目录与本说明文件，不继续平铺新增脚本。
- 新脚本必须按功能归类到对应子目录；若现有子目录都不合适，再新增新的功能目录。
- 脚本路径变更时，必须同步更新：
  - `README.md`
  - 测试中的脚本路径引用
  - 脚本之间的相互调用路径

## 3. 开发约束

- Shell 脚本默认使用仓库根目录相对路径调用其他仓库脚本，避免写死用户本机绝对路径。
- Python 脚本保持单一职责，复用核心逻辑时优先委托到 `src/agent_kit/` 中的可测试模块。
- 改脚本行为时，优先补对应测试，再调整实现。
