# self-evolve 详细中文说明文档

## 一、概述

`self-evolve` 是 [agent-kit](https://github.com/Mikko-ww/agent-kit) 平台的第一方插件，提供**以项目为基础的自我进化能力**。它能够以 **Agent Skill** 的方式接入 Cursor、VS Code Copilot（GitHub Copilot）、Codex 等主流编码 Agent，让 Agent 在使用过程中通过无感或询问方式积累开发经验，并将这些经验自动进化为项目级的开发规则，最终同步到统一的 Skill 文件中，供所有 Agent 发现和使用。

### 核心理念

```
经验捕获 → 模式识别 → 规则推广 → Agent Skill 同步
```

每个项目都有自己独立的进化数据和规则，统一存储在项目根目录 `.agents/` 中，不按智能体类型拆分。当 Agent 在开发过程中发现重复的模式或重要的经验时，可以通过 `capture` 命令记录下来。当同一模式被多次发现后，系统会自动将其推广为永久规则，并同步到 `.agents/skills/self-evolve/SKILL.md`，任何支持技能发现的 Agent 都可以自动使用。

## 二、安装

### 前置条件

- Python >= 3.11
- [agent-kit](https://github.com/Mikko-ww/agent-kit) 已安装

### 安装命令

```bash
agent-kit plugins install self-evolve
```

## 三、快速开始

### 步骤 1：初始化项目

```bash
cd your-project
agent-kit self-evolve init
```

初始化向导会引导你：
1. 设置推广门槛参数

初始化完成后，会在项目中创建：
- `.agents/self-evolve/` 目录（存放配置、学习数据和规则）
- `.agents/skills/self-evolve/SKILL.md`（统一的 Skill 文件）

### 步骤 2：捕获学习

在开发过程中发现重要经验时，使用 `capture` 命令记录：

```bash
agent-kit self-evolve capture \
  --summary "启动前必须验证所有环境变量" \
  --domain debugging \
  --priority high \
  --pattern-key env-var-validation \
  --task-id task-42
```

参数说明：
| 参数 | 说明 | 必填 |
|------|------|------|
| `--summary, -s` | 学习摘要 | 是 |
| `--domain, -d` | 领域标签 | 是 |
| `--priority, -p` | 优先级（low/medium/high/critical） | 否（默认 medium） |
| `--detail` | 详细描述 | 否 |
| `--action` | 建议操作 | 否 |
| `--pattern-key` | 模式键，用于关联相似条目 | 否 |
| `--task-id` | 关联任务 ID | 否 |
| `--tags` | 逗号分隔的标签 | 否 |

### 步骤 3：分析模式

```bash
agent-kit self-evolve analyze
```

分析命令会：
1. 扫描所有学习条目
2. 通过 `pattern_key` 识别相似模式
3. 自动交叉链接相关条目
4. 列出满足推广条件的候选条目

### 步骤 4：推广为规则

```bash
agent-kit self-evolve promote L-20260330-001 \
  --rule "在应用启动时验证所有必需的环境变量"
```

手动推广一条学习为永久规则。推广前会检查：
- 重复次数 ≥ 推广门槛
- 出现在 ≥ 最少任务数的不同任务中

### 步骤 5：同步到 Agent

```bash
agent-kit self-evolve sync
```

将所有推广的规则同步到统一的 Skill 文件中。

### 步骤 6：一键进化

```bash
agent-kit self-evolve evolve
```

**一键完成完整进化循环**：
1. 分析所有学习条目的模式
2. 自动推广满足条件的条目为规则
3. 将规则同步到 Skill 文件

这是最常用的命令，建议在每次开发会话结束后运行。

## 四、完整命令参考

| 命令 | 说明 |
|------|------|
| `init` | 为当前项目初始化自我进化 |
| `capture` | 捕获一条新的学习记录 |
| `list` | 列出学习记录（支持过滤） |
| `analyze` | 分析学习模式并检测重复项 |
| `promote` | 将学习记录推广为永久规则 |
| `sync` | 将推广规则同步到统一 Skill 文件 |
| `evolve` | 一键进化：分析 + 自动推广 + 同步 |
| `status` | 显示进化状态概览 |
| `search` | 按领域、标签或关键词搜索推广规则 |

### 查看帮助

```bash
agent-kit self-evolve --help
agent-kit self-evolve capture --help
```

### 列出学习记录

```bash
# 列出所有活跃记录
agent-kit self-evolve list --status active

# 按领域过滤
agent-kit self-evolve list --domain debugging

# 按优先级过滤
agent-kit self-evolve list --priority high

# 限制显示数量
agent-kit self-evolve list --limit 10
```

### 查看进化状态

```bash
agent-kit self-evolve status
```

输出示例：
```
Total learnings: 15
  active: 8
  promoted: 5
  resolved: 2
Promoted rules: 5
Active domains: debugging, testing, architecture
```

### 搜索规则

```bash

# 按标签搜索

# 模糊关键词搜索
agent-kit self-evolve search --keyword "环境变量"

# 显示域统计
agent-kit self-evolve search --stats

# 显示详细规则内容
agent-kit self-evolve search --domain debugging --detail
```

## 五、Agent Skill 集成详解

### 5.1 自适应分层策略

同步模块采用类似「图书馆找书」的自适应分层策略，根据规则数量自动选择最优输出方式：

| 策略 | 条件 | 输出 | 类比 |
|--------|------|------|------|
| **inline** | 规则数 ≤ `inline_threshold`（20） | 所有规则内联到 SKILL.md | 书架上直接看 |
| **index** | 规则数 > `inline_threshold` | SKILL.md 仅含索引表，详细拆分到分域文件 | 查目录再找书 |

### 5.2 输出文件

| 输出文件 | 说明 | 生成条件 |
|---------|------|----------|
| `.agents/skills/self-evolve/SKILL.md` | 统一 Skill 文件（含 YAML frontmatter） | 始终生成 |
| `.agents/skills/self-evolve/catalog.json` | 结构化规则目录 | index 策略 |
| `.agents/skills/self-evolve/domains/*.md` | 分域详细规则文件 | index 策略 |
| `.agents/skills/self-evolve/find_rules.py` | 零依赖本地搜索脚本 | 始终同步 |

不按智能体类型（Cursor/Copilot/Codex）拆分配置或输出文件。任何支持 `.agents/skills/` 技能发现机制的 Agent 都可以自动发现并使用此 Skill。

### 5.2 Skill 文件内容

`SKILL.md` 文件采用标准 Agent Skill 格式，包含：

1. YAML frontmatter（`name` 和 `description`）
2. 规则概览和使用指南
3. 规则内容（inline 模式）或规则索引表（index 模式）
4. 引导 Agent 发现新模式时调用 `capture` 的指令

示例输出（inline 策略）：

```markdown
---
name: self-evolve
description: 项目自进化规则 - 由开发经验自动沉淀生成
---

# 项目自进化规则

## 规则

### debugging

- **R-001** 在应用启动时验证所有必需的环境变量
  > 标签：env, validation

...
```

### 5.3 无感进化体验

生成的 Skill 文件中包含引导指令，告诉 Agent：

> "当你在开发过程中发现重复模式或重要经验时，调用 `agent-kit self-evolve capture` 记录下来。"

这意味着 **Agent 在正常工作时就会主动捕获学习记录**，实现无感的自我进化循环。

## 六、项目数据结构

### 6.1 目录布局

```
<项目根目录>/
├── .agents/
│   ├── self-evolve/                 # 自我进化数据目录
│   │   ├── config.jsonc             # 项目级配置
│   │   ├── learnings/               # 学习条目存储
│   │   │   ├── L-20260330-001.jsonc
│   │   │   ├── L-20260330-002.jsonc
│   │   │   └── ...
│   │   └── rules.jsonc              # 推广规则
│   └── skills/
│       └── self-evolve/
│           ├── SKILL.md             # 统一 Skill 文件（自动生成）
│           ├── catalog.json         # 规则目录（index 策略）
│           ├── find_rules.py        # 本地搜索脚本
│           └── domains/             # 分域详细规则（index 策略）
```

### 6.2 配置文件

文件位置：`.agents/self-evolve/config.jsonc`

```jsonc
{
  // 插件标识
  "plugin_id": "self-evolve",
  // 配置版本（当前为 3）
  "config_version": 3,
  // 推广门槛：学习条目需要至少重复出现几次
  "promotion_threshold": 3,
  // 推广时间窗口（天）
  "promotion_window_days": 30,
  // 推广最少任务数：学习需要出现在几个不同任务中
  "min_task_count": 2,
  // 是否在 evolve 时自动推广
  "auto_promote": false,
  // 自适应策略阈值（规则数 ≤ 此值时采用 inline，> 此值时采用 index）
  "inline_threshold": 20
}
```

### 6.3 学习条目

文件位置：`.agents/self-evolve/learnings/L-YYYYMMDD-NNN.jsonc`

```jsonc
{
  "id": "L-20260330-001",
  "timestamp": "2026-03-30T12:00:00+00:00",
  "priority": "high",
  "status": "active",
  "domain": "debugging",
  "summary": "启动前必须验证所有环境变量",
  "detail": "多次发现应用启动失败是因为环境变量缺失",
  "suggested_action": "添加启动脚本中的环境变量检查",
  "pattern_key": "env-var-validation",
  "see_also": ["L-20260328-003"],
  "recurrence_count": 3,
  "task_ids": ["task-42", "task-56"],
  "metadata": {}
}
```

字段说明：

| 字段 | 说明 |
|------|------|
| `id` | 唯一标识，格式 `L-YYYYMMDD-NNN` |
| `timestamp` | 创建时间（ISO 8601） |
| `priority` | 优先级：`low` / `medium` / `high` / `critical` |
| `status` | 状态：`active` / `resolved` / `promoted` / `promoted_to_skill` |
| `domain` | 领域标签 |
| `summary` | 简要摘要 |
| `detail` | 详细描述 |
| `suggested_action` | 建议操作 |
| `pattern_key` | 模式键，用于关联相似条目 |
| `see_also` | 关联条目 ID 列表 |
| `recurrence_count` | 重复次数 |
| `task_ids` | 出现在哪些任务中 |

### 6.4 推广规则

文件位置：`.agents/self-evolve/rules.jsonc`

```jsonc
{
  "rules": [
    {
      "id": "R-001",
      "source_learning_id": "L-20260330-001",
      "rule": "在应用启动时验证所有必需的环境变量",
      "domain": "debugging",
      "created_at": "2026-03-30T15:00:00+00:00",
      "tags": ["env", "validation"],
      "title": "环境变量校验"
    }
  ]
}
```

## 七、进化流程详解

### 7.1 完整进化循环

```
1. Agent 在开发中发现经验
   ↓
2. 调用 capture 记录学习条目
   ↓
3. 同一模式被多次记录（pattern_key 匹配）
   ↓
4. analyze 识别模式组，标记推广候选
   ↓
5. promote 将候选提炼为永久规则
   ↓
6. sync 将规则写入统一的 Skill 文件
   ↓
7. Agent 在后续开发中遵循已进化的规则
   ↓
   （循环回到步骤 1）
```

### 7.2 推广条件

一条学习要被推广为规则，必须同时满足：

1. **重复次数**：`recurrence_count >= promotion_threshold`（默认 ≥ 3）
2. **跨任务出现**：`len(task_ids) >= min_task_count`（默认 ≥ 2）
3. **状态限制**：不能已经是 `promoted` 或 `promoted_to_skill`

### 7.3 一键进化 vs 手动进化

**一键进化**（推荐）：
```bash
agent-kit self-evolve evolve
```
自动完成 analyze → auto-promote → sync 全流程。

**手动进化**（精细控制）：
```bash
# 先分析
agent-kit self-evolve analyze

# 手动推广指定条目
agent-kit self-evolve promote L-20260330-001 --rule "自定义规则描述"

# 手动同步
agent-kit self-evolve sync
```

## 八、典型使用场景

### 场景 1：Cursor 中的无感进化

1. 在 Cursor 中打开已初始化的项目
2. 通过 skills-link 将 `.agents/skills/` 链接到 Cursor 的技能目录
3. 当 Cursor 发现重复模式时，会按照 Skill 中的引导调用 `capture`
4. 开发者定期运行 `evolve` 完成进化循环
5. 新规则自动更新到 Skill 文件中

### 场景 2：多 Agent 协作的项目进化

1. 所有 Agent 通过 `.agents/skills/self-evolve/SKILL.md` 共享同一份进化规则
2. 无论使用 Cursor、Copilot 还是 Codex，都能发现并遵循已进化的规则
3. 任何 Agent 都可以通过终端捕获新的学习
4. 运行 `evolve` 后，统一的 Skill 文件自动更新

### 场景 3：团队协作中的知识共享

1. `.agents/self-evolve/config.jsonc` 和 `.agents/self-evolve/rules.jsonc` 可以提交到 Git
2. 团队成员共享已进化的规则
3. 每个成员的 Agent 都能使用项目积累的经验

## 十、版本信息

- 当前版本：**0.3.0**
- API 版本：1

- 要求 agent-kit core ≥ 0.1.0

| 特性 | 0.2.x | 0.3.0 |
|------|-------|-------|
| 数据存储位置 | `.self-evolve/` | `.agents/self-evolve/` |

## 十一、常见问题

### Q: 初始化后 `.agents/self-evolve/` 目录应该提交到 Git 吗？

**推荐做法**：
- `.agents/self-evolve/config.jsonc` — **提交**（共享项目配置）
- `.agents/self-evolve/rules.jsonc` — **提交**（共享进化规则）
- `.agents/self-evolve/learnings/` — **可选**（个人学习记录，可按需决定）

### Q: 生成的 Skill 文件应该提交吗？

建议提交 `.agents/skills/self-evolve/SKILL.md`，这样团队成员可以直接使用已进化的规则。

### Q: 一键进化 `evolve` 命令做了什么？

`evolve` = `analyze` + 自动推广满足条件的学习 + `sync`。等效于：

```bash
agent-kit self-evolve analyze
# 自动推广所有候选条目
agent-kit self-evolve sync
```
